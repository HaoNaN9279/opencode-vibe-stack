# OpenCode Vibe Coding 工具链使用指南

## 一、部署

### 1.1 前置条件

- 已安装 [OpenCode v2.1+](https://opencode.ai)
- 已安装 Git

### 1.2 克隆仓库

```bash
git clone <仓库地址> ~/opencode-vibe-stack
cd ~/opencode-vibe-stack
```

### 1.3 一键部署

**Windows**（以管理员身份运行 PowerShell）：

```powershell
.\scripts\deploy.ps1
```

会做三件事：
1. 设置环境变量 `OPCODE_STACK_ROOT`
2. 在 OpenCode 配置目录（`%APPDATA%\OpenCode\User`）下创建符号链接
3. 配置 npm/pip/dotnet 国内镜像

**Linux / WSL2**：

```bash
chmod +x scripts/deploy.sh
./scripts/deploy.sh
```

### 1.4 手动部署（不想跑脚本时）

设置环境变量：

```bash
# Linux/macOS
export OPCODE_STACK_ROOT=~/opencode-vibe-stack

# Windows PowerShell
$env:OPCODE_STACK_ROOT = "D:\dev\opencode-vibe-stack"
```

创建符号链接（以 Linux 为例）：

```bash
mkdir -p ~/.config/opencode/User
ln -sfn ~/opencode-vibe-stack/core ~/.config/opencode/User/core
ln -sfn ~/opencode-vibe-stack/domains ~/.config/opencode/User/domains
ln -sfn ~/opencode-vibe-stack/combinations ~/.config/opencode/User/combinations
```

### 1.5 验证部署

```bash
opencode --version
```

重启 OpenCode 后配置即生效。

## 二、在 OpenCode 中使用

### 2.1 创建新项目

```bash
# Windows
.\scripts\new-project.ps1 -ProjectName MyGame -Template unity
```

或手动创建项目目录，在项目根目录添加 `.opencode/workspace.opencode.md`：

```yaml
---
version: "2.1"
name: "MyGame"
type: "unity-project"

imports:
  - "${OPCODE_STACK_ROOT}/domains/game-engine/unity/csharp-api/module.opencode.md"

agents:
  default: "unity.csharp.orchestrator"
---
```

### 2.2 多领域组合项目

如果项目同时使用 Unity + Blender + Node.js：

```yaml
---
version: "2.1"
name: "MyGameToolchain"
type: "multi-domain"

domains:
  - "game-engine.unity.csharp-api"
  - "dcc.blender.python-api"
  - "web.nodejs.express"

# 或直接用预定义的组合包
imports:
  - "${OPCODE_STACK_ROOT}/combinations/game-asset-pipeline/combination.opencode.md"

agents:
  default: "game-asset-pipeline.orchestrator"
---
```

### 2.3 在 Chat 中使用

部署完成后，直接在 OpenCode 对话中使用即可：

> "帮我创建一个 Unity 角色控制器组件"
> "写一个 Blender 插件，批量导出 FBX"
> "做一个 Express REST API，用户增删改查"

系统会自动加载对应领域的 Agent 和技能来处理。

## 三、后期维护

### 3.1 更新工具链

```bash
cd $OPCODE_STACK_ROOT
git pull origin main
```

### 3.2 只更新特定模块

```powershell
.\scripts\update-stack.ps1 -Module game-engine.unity.csharp-api
```

### 3.3 在另一台机器上部署

```bash
git clone <仓库地址>
./scripts/deploy.sh    # 或 deploy.ps1
```

五分钟内即可恢复完整环境。

## 四、扩展（添加新模块）

### 4.1 快速添加新 API 层

在对应领域目录下创建子目录和 `module.opencode.md`：

```
domains/<领域>/<工具>/<API层>/
├── module.opencode.md
├── agents/
├── skills/
├── rules/
├── templates/
└── mcp/
```

### 4.2 module.opencode.md 模板

```yaml
---
version: "2.1"
module:
  id: "领域.工具.API层"        # 唯一标识
  name: "显示名称"
  description: "描述"

  dependencies:
    - "../../../core/config.opencode.md"

  provides:
    languages: ["语言1"]
    apis: ["API名称"]

imports:
  - "./skills/*.opencode.md"
  - "./rules/*.opencode.md"
  - "./templates/*.opencode.md"
---
```

### 4.3 添加新 Agent

在对应模块的 `agents/` 目录下创建 `*.opencode.md`：

```yaml
---
version: "2.1"
agent:
  id: "领域.工具.API层.agent-name"
  name: "显示名称"

  a2a:
    enabled: true
    roles: ["角色"]
    capabilities: ["能力1", "能力2"]
    handles_tasks: ["可处理的任务"]
    depends_on: ["其他Agent ID"]

  persona: |
    你是一位怎样的专家。
---
```

### 4.4 添加新 Skill

在对应模块的 `skills/` 目录下创建 `*.opencode.md`，代码模板放在 `---` 下方。

## 五、目录速查

| 路径 | 用途 |
|------|------|
| `core/` | 全局配置、A2A协议、通用技能/规则 |
| `platforms/` | 平台配置（Windows/Linux/WSL2） |
| `domains/dcc/` | DCC软件（Blender/Houdini/Maya） |
| `domains/game-engine/` | 游戏引擎（Unity/Unreal） |
| `domains/desktop/` | 桌面开发（C#/C++/Python） |
| `domains/web/` | Web开发（Node.js/React/Vue） |
| `combinations/` | 预定义的跨领域组合包 |
| `scripts/` | 部署和项目初始化脚本 |
| `workspace-templates/` | 项目模板 |

# OpenCode Photoshop工具与插件开发支持扩展方案（无MCP版）
本方案基于OpenCode原生的**Rules、Commands、Skills、Agents**四层架构设计，完全不依赖任何MCP插件，针对Photoshop三大开发方向（ExtendScript/JSX脚本、UXP统一扩展平台、C++ SDK底层插件）提供全生命周期支持，重点解决开发者最痛点的API记忆、调试困难、版本兼容和性能优化问题。

## 一、核心Rules扩展（基础能力层）
Rules定义了OpenCode如何识别、解析和验证Photoshop相关代码，是所有上层功能的基础。

### 1. 语言与文件类型识别规则
- **文件类型映射**：
  - `.jsx`/`.jsxinc` → ExtendScript脚本（ECMAScript 3 + Adobe扩展）
  - `.psjs` → Photoshop UXP专用脚本
  - `manifest.json`（包含`"manifestVersion"`且`"host.app"`包含`"PS"`）→ UXP插件配置
  - `.cpp`/`.h`（包含`#include "PISDK.h"`或`#include "PSPlugIn.h"`）→ Photoshop C++ SDK插件
  - `.atn` → Photoshop动作文件（支持解析和代码转换）

- **语法解析规则**：
  - 为ExtendScript添加专用语法解析器，正确识别其与现代JavaScript的差异（无`let`/`const`、无箭头函数、特殊全局对象、`$`前缀变量、`#include`指令等）
  - 为UXP manifest.json添加多版本JSON Schema验证（v4、v5、v6），严格检查Adobe要求的必填字段和格式
  - 识别Photoshop C++ SDK的特殊宏定义和数据类型（如`SPBasicSuite`、`PIActionDescriptor`）

### 2. 代码风格与规范规则
- 基于Adobe官方开发指南，定义Photoshop ExtendScript和UXP插件的代码风格标准
- 强制检查：
  - UXP代码必须使用`"use strict"`严格模式
  - ExtendScript中禁止直接使用`app.activeDocument`（必须先检查`app.documents.length > 0`）
  - UXP插件权限声明最小化原则，标记过度授权的权限项
  - C++插件的内存管理规则（必须释放`PIActionDescriptor`等对象）

### 3. 静态错误检测规则
- **通用错误检测**：
  - 未处理用户取消操作（`if (result == null) return`）
  - 越界访问图层/通道/路径集合
  - 使用已废弃的API（如`app.executeAction`的旧参数格式）
  - 循环中频繁调用`app.doScript`导致性能下降

- **UXP专属错误检测**：
  - 未申请必要权限就调用敏感API（如文件系统访问、网络请求）
  - 在主线程中执行耗时操作导致UI卡顿
  - manifest.json中`version`与`host.minVersion`不兼容

- **版本兼容性检测**：
  - 标记在指定Photoshop版本（2020-2026）中不可用的API
  - 提示不同版本间API行为的差异（如`layer.name`在PS 2025中的编码变化）

### 4. 项目结构规则
- 定义三种标准Photoshop项目的目录结构：
  - ExtendScript脚本项目：`/scripts`、`/includes`、`/docs`
  - UXP插件项目：`/src`、`/dist`、`/public`、`manifest.json`
  - C++ SDK插件项目：`/source`、`/include`、`/projects`（VS/Xcode）、`/resources`

## 二、核心Commands扩展（用户交互层）
Commands是用户可以直接调用的原子操作，覆盖从项目创建到发布的完整流程。

### 1. 项目创建与初始化
```bash
# 创建基础ExtendScript脚本项目
opencode create photoshop-script --name "BatchResize" --version "1.0.0"

# 创建UXP插件项目（支持面板/脚本/无头三种类型）
opencode create photoshop-uxp --name "LayerManager" --type panel --framework react

# 创建C++ SDK插件项目（自动配置VS 2022/Xcode 15）
opencode create photoshop-cpp --name "CustomFilter" --sdk-version "2025"

# 在现有目录初始化Photoshop开发环境
opencode init photoshop --type uxp
```

### 2. 代码生成
```bash
# 生成常用图层操作代码
opencode generate photoshop-layer-operation --action create,rename,delete,duplicate

# 生成批量处理文件模板
opencode generate photoshop-batch-process --input-folder --output-format jpg,png

# 生成UXP插件UI组件（按钮、滑块、颜色选择器等）
opencode generate photoshop-uxp-ui --component button,slider,color-picker

# 生成完整的manifest.json配置文件
opencode generate photoshop-manifest --version "5.0.0" --host-min "24.0" --permissions "fileSystem,network"

# 将Photoshop动作(.atn)转换为ExtendScript代码
opencode generate photoshop-action-to-script --input "MyAction.atn"
```

### 3. 运行与调试
```bash
# 直接在Photoshop中运行当前打开的ExtendScript脚本
opencode run photoshop-script --target "Photoshop 2025"

# 启动UXP调试器并连接到Photoshop
opencode debug photoshop-uxp --port 9222

# 附加到正在运行的Photoshop实例进行调试
opencode attach photoshop --pid 12345

# 实时查看Photoshop脚本日志和UXP插件日志
opencode log photoshop --follow --level error,warn
```

### 4. 打包与发布
```bash
# 将UXP插件打包为ZXP格式
opencode package photoshop-uxp --output "LayerManager.zxp" --compress high

# 对ZXP文件进行数字签名（支持自签名和Adobe官方签名）
opencode sign photoshop-zxp --input "LayerManager.zxp" --certificate "my-cert.p12"

# 生成Adobe Marketplace发布所需的所有文件
opencode prepare-marketplace photoshop --output "./marketplace-package"

# 清理项目中的临时文件和构建产物
opencode clean photoshop-project
```

### 5. 工具与辅助
```bash
# 打开Photoshop API文档并搜索指定内容
opencode docs photoshop api --query "layer.applyFilter"

# 检查代码在指定Photoshop版本中的兼容性
opencode check-compatibility photoshop --versions "2023,2024,2025"

# 将ExtendScript代码转换为UXP兼容代码（提供详细转换建议）
opencode convert extendscript-to-uxp --input "old-script.jsx" --output "new-script.psjs"
```

## 三、核心Skills扩展（智能能力层）
Skills是组合多个Commands和Rules形成的高级智能能力，是提升开发效率的关键。

### 1. 上下文感知的智能代码补全
- 提供完整的Photoshop ExtendScript、UXP和C++ SDK API补全
- 上下文感知：在处理图层时自动提示图层相关方法，处理文档时提示文档相关属性
- 版本感知：只显示当前目标Photoshop版本支持的API
- 代码片段建议：根据用户输入自动推荐常用代码片段（如批量重命名图层、导出为Web格式）

### 2. 实时错误诊断与自动修复
- 实时检测代码中的错误和潜在问题，提供详细的错误说明和修复建议
- 一键自动修复常见错误：
  - 添加缺失的UXP权限声明
  - 修复manifest.json中的配置错误
  - 替换已废弃的API为最新版本
  - 添加必要的空值检查
- 提供调试步骤建议，帮助用户快速定位问题

### 3. Photoshop专属代码优化
- **性能优化**：
  - 识别循环中低效的API调用，建议使用更高效的替代方案（如用`layers.everyItem()`代替for循环）
  - 建议使用`app.doScript`包装批量操作以提高执行速度
  - 提示避免在循环中创建不必要的对象
- **代码现代化**：将旧的ExtendScript代码转换为更现代的UXP代码风格
- **代码简化**：简化复杂的Photoshop操作代码，提高可读性和可维护性

### 4. 版本迁移支持
- 提供从ExtendScript到UXP的完整迁移指南和代码转换工具
- 自动更新旧版本UXP manifest.json到最新格式
- 识别并替换已废弃的UXP API
- 提供从旧版本C++ SDK到新版本的迁移建议

### 5. 文档自动生成
- 基于代码注释自动生成Photoshop插件的API文档
- 自动生成插件的用户文档，包括功能介绍、使用方法和截图说明
- 保持文档与代码的同步，当代码修改时自动更新文档

## 四、核心Agents扩展（自主代理层）
Agents是能够独立完成复杂任务的自主代理，提供端到端的开发支持。

### 1. Photoshop插件开发全流程助手
- 全程指导用户从需求分析到发布的整个开发流程
- 根据用户需求推荐最合适的插件类型（ExtendScript适合快速脚本，UXP适合现代UI插件，C++适合高性能滤镜）
- 帮助用户规划项目结构和功能模块
- 解答开发过程中遇到的各种问题，提供最佳实践建议
- 跟踪开发进度，提醒用户完成必要的步骤

### 2. Photoshop代码审查代理
- 自动审查Photoshop插件的代码质量、性能和安全性
- 检查代码是否符合Adobe官方开发规范
- 识别潜在的bug、内存泄漏和安全漏洞
- 提供详细的审查报告和改进建议
- 支持自定义审查规则，满足团队特定需求

### 3. Photoshop调试助手
- 分析调试日志和错误信息，自动定位问题根源
- 提供详细的调试步骤和解决方案
- 模拟不同的Photoshop环境和用户场景，帮助用户复现问题
- 收集常见问题的解决方案，形成知识库供用户查询

### 4. Photoshop性能优化代理
- 分析脚本的运行性能，生成详细的性能报告
- 识别性能瓶颈，提供针对性的优化建议
- 自动优化代码中的性能问题（如替换低效的API调用）
- 提供性能测试工具，帮助用户验证优化效果
- 对比不同优化方案的性能差异

### 5. Photoshop插件发布助手
- 检查插件是否符合Adobe Creative Cloud Marketplace的所有发布要求
- 帮助用户准备发布所需的所有文件和资料（图标、截图、描述、发布说明等）
- 生成符合要求的营销材料
- 指导用户完成发布流程
- 提供发布后的监控和维护建议

## 五、可扩展的高级功能
除了上述核心组件外，还可以扩展以下功能进一步提升开发体验：

### 1. 官方API文档深度集成
- 将Photoshop官方API文档完整集成到OpenCode中，支持离线查询和搜索
- 提供API的使用示例和最佳实践
- 显示API的版本兼容性信息和变更历史
- 直接从代码跳转到对应的API文档

### 2. 代码片段库与模板中心
- 内置数百个常用的Photoshop脚本代码片段
- 提供各种类型的插件模板（批量处理工具、图层管理工具、滤镜插件等）
- 支持用户自定义代码片段和模板
- 分享和下载社区贡献的代码片段和模板

### 3. 与Photoshop的实时交互
- 直接从OpenCode运行脚本到Photoshop，无需手动复制粘贴
- 实时同步Photoshop中的状态到OpenCode（当前打开的文档、选中的图层、历史记录等）
- 在OpenCode中预览UXP插件的UI效果
- 支持断点调试，在OpenCode中设置断点，在Photoshop中触发执行

### 4. 自动化测试框架
- 提供Photoshop插件的自动化测试框架
- 支持单元测试和集成测试
- 模拟不同的Photoshop环境和用户操作
- 生成详细的测试报告

## 六、优先级建议
1. **最高优先级**：UXP插件开发支持（manifest验证、API补全、调试、打包）
2. **高优先级**：ExtendScript脚本开发支持（代码补全、错误检测、运行）
3. **中优先级**：性能优化工具、版本兼容性检查
4. **低优先级**：C++ SDK支持、自动化测试框架
